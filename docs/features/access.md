# 访问码

> 仅在桌面版设置中配置，不提供 CLI 选项

给浏览器端的阅览页面加一道**访问码**门禁：访客先输入正确的访问码才能看到内容。适合在内网或临时分享给同事、把文档交给评审方时，挡住无关的眼睛。

::: warning 这是应用层的访问控制，不是传输加密
访问码只在 Markon 这一层校验「能不能看」，本身不加密链路。真要把服务暴露到公网，仍需在前面套 HTTPS / 反向代理，详见[反向代理](/advanced/reverse-proxy)。
:::

## 两级访问码

Markon 的访问码分两级，各管各的范围：

- **全局访问码（服务级）** —— 在「全局」选项卡设置（标签为「全局默认认证码」）。
- **工作区访问码** —— 在「工作区」选项卡，点每张工作区卡片上的**锁图标**单独设置。

两者的关系是「就近覆盖」：

| 工作区是否设了自己的码 | 全局是否设了码 | 进入该工作区需要 |
|--------------------|-------------|----------------|
| 设了 | 任意 | **该工作区自己的码**（全局码解锁不了它） |
| 没设 | 设了 | 全局码 |
| 没设 | 没设 | 无门禁（开放访问） |

也就是说：**工作区自己的码会覆盖全局码**。全局码只负责那些「没有自己专属码」的工作区；一个设了专属码的工作区，必须用它自己的码，拿全局码解不开，反之亦然。两边都没设，则完全不设门禁。

## 设置访问码

访问码**只能在桌面版设置里改**，没有对应的命令行选项。

### 全局码（「全局」选项卡）

在「全局默认认证码」一栏填入新码并点「应用」即可；清空输入框再应用则移除全局码。

> 改动全局码会**重启服务**以让运行中的实例生效——重启期间链接会短暂中断。

### 工作区码（「工作区」选项卡）

点目标工作区卡片上的**锁图标**展开输入框，填入码、确认即可；清空则移除该工作区的码。

> 工作区码是**即时生效**的，不需要重启服务。

## 访客如何解锁

当某个范围设了访问码，访客在浏览器里访问对应工作区时会先看到一张**「访问码」门禁页**：

1. 输入框旁有一个**按住显示**的眼睛图标——按住时明文显示已输入的码，松开即重新遮蔽，方便核对又不会把码留在屏幕上。
2. 输入正确的码并解锁后，Markon 会写入一个**签名 Cookie**，有效期约 **30 天**，在此期间该范围自动放行，无需反复输入。
3. Cookie 按范围区分：解锁了全局范围，并不会顺带解开某个设了专属码的工作区，反之亦然。

> Cookie 用每台机器的持久盐签名，因此**重启服务后仍然有效**，访客不必重新解锁。连续输错会触发按来源 IP 的冷却，逐步拉长重试间隔，以减缓暴力猜测。

## 局限

- 访问码是**应用层**的「看不看得到」控制，不替代 HTTPS / 反向代理等传输层安全措施。
- 没有命令行开关——临时用 CLI 起的服务若需门禁，需借助前置的反向代理（参见[反向代理](/advanced/reverse-proxy)）做鉴权。
